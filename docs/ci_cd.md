```markdown
# CI/CD Workflow

Проект использует GitHub Actions для автоматизации проверки качества кода и тестирования.  

## Основные задачи workflow

1. **Автоматический запуск**
   - По расписанию: каждое воскресенье в 00:00 UTC
   - На каждом push и pull request в ветку `main`

2. **Шаги workflow**
   - **Checkout** — клонирование репозитория
   - **Установка Python** — версия 3.12
   - **Установка зависимостей** — из `requirements.txt`
   - **Запуск тестов** — с помощью `pytest`, генерация отчета `results.xml`
   - **Загрузка результатов тестов** — как артефакт
   - **Lint кода** — с помощью `flake8`
   - **Загрузка отчета lint** — как артефакт

3. **Используемые инструменты**
   - `pytest` — для unit-тестов
   - `flake8` — для статического анализа кода
   - `actions/upload-artifact` — для сохранения отчетов
   - `cron` — для scheduled workflow

## Преимущества CI/CD
- Автоматическая проверка качества кода
- Возможность быстрого обнаружения ошибок
- Сохранение отчетов для анализа
```
